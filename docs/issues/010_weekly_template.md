Labels sugeridas: good first issue, help wanted, community, P1

## Contexto
El post semanal necesita una plantilla consistente para evitar PII y exigir evidencia.

## Objetivo
Crear una plantilla de publicación semanal + checklist de publicación.

## Alcance
Incluye:
- Plantilla Markdown reusable
- Checklist previa a publicar (sin PII, claims con evidencia)
- Sección de “feedback solicitado”

No incluye:
- Autopublicación
- Métricas de engagement

## Criterios de aceptación
- [ ] Plantilla breve y clara
- [ ] Checklist con verificación de evidencia
- [ ] Enlace desde `docs/README.md`

## Cómo verificar
1) Usar la plantilla para un post ficticio
2) Confirmar que el checklist es accionable

## Pistas de implementación
- Basarse en el formato de community_draft
